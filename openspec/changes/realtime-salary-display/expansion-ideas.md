# 拓展型玩法功能 - 想法收集

本文档记录实时工资展示页面的潜在拓展方向，供后续迭代参考。

---

## 1. 游戏化 (Gamification)

利用游戏机制提升用户参与度和持续使用动力。

### 1.1 成就系统

| 成就名称 | 触发条件 | 奖励 |
|---------|---------|------|
| 坚持打卡 | 连续使用N天 | 徽章 + 特殊主题 |
| 里程碑达成 | 累计赚满N元 | 庆祝动画 |
| 夜猫子 | 连续N天在深夜工作 | 夜猫徽章 |
| 早鸟 | 连续N天早起工作 | 早鸟徽章 |
| 加班战士 | 单周加班超过N小时 | 战士徽章 |

### 1.2 进度条与目标

- **今日目标**：设置每日目标金额，实时显示进度条
- **本周目标**：周末查看本周达成情况
- **攒钱目标**：用户设定大目标（如买房、旅游），实时显示累计进度

### 1.3 随机激励

- 工作中随机弹出激励语（"再坚持一小时就能买杯奶茶了！"）
- 里程碑时刻触发特殊效果（赚满1000元时金币雨）

## 2. 社交互动 (Social)

通过社交元素增强用户之间的连接和互相激励。

### 2.1 分享功能

- **截图分享**：一键生成带激励语的精美图片
- **里程碑分享**：达成目标时生成分享卡片
- **定制语录**：用户可自定义分享时的激励语

### 2.2 团队模式

- 组队功能：和小伙伴组成"赚钱小队"
- 实时看到队友也在赚钱（不显示具体金额，只显示状态）
- 团队里程碑：团队累计达成某目标时集体庆祝

### 2.3 排行榜（谨慎使用）

> ⚠️ 排行榜可能产生负面效果（焦虑、比较压力），需谨慎设计

可选方案：
- 只显示自己排名区间，不显示具体名次
- 匿名排行，只显示百分比位置
- 完全不使用排行榜

### 2.4 激励墙

- 用户可提交自己的激励语录
- 随机展示其他用户的激励语
- 点赞机制筛选优质语录

## 5. 薅羊毛玩法 (Wool-Gathering)

> ⭐ **创意亮点**：反向思维——不是"努力赚钱"，而是"薅公司羊毛赚钱"
> 幽默感强，用户共鸣度高，"我摸鱼也是在赚公司的钱"

### 5.1 水费薅羊毛

在公司喝水 = 赚公司的水费

**计算公式**：
```
赚的钱 = 喝水量(ml) ÷ 1,000,000 × 水费单价(元/吨)
```

**示例**：
- 喝了 500ml 水，水费 5元/吨 → 赚了 ¥0.0025（0.25分）
- 喝了 2000ml 水（一个大保温杯），水费 5元/吨 → 赚了 ¥0.01（1分）

**交互设计**：
- 快捷按钮：+100ml、+250ml、+500ml、自定义输入
- 今日累计喝水量显示
- 累计"赚"的水费显示

### 5.2 电费薅羊毛

在公司给设备充电 = 赚公司的电费

**计算公式**：
```
赚的钱 = 充电量(度) × 电费单价(元/度)
```

**设备电量估算表**（供用户参考）：

| 设备 | 每次充满约耗电 |
|-----|---------------|
| 手机 | ~0.01度（10Wh） |
| 平板 | ~0.03度（30Wh） |
| 笔记本电脑 | ~0.05度/小时（50W充电功率） |
| 蓝牙耳机 | ~0.001度（1Wh） |
| 智能手表 | ~0.0005度（0.5Wh） |

**交互设计**：
- 快捷按钮：充了一次手机、充了一次耳机、笔记本充电1小时...
- 自定义输入充电度数
- 今日累计充电量显示
- 累计"赚"的电费显示

### 5.3 摸鱼时间薅羊毛

上厕所、下楼溜达 = 公司付工资但你没干活 = 白赚这段时间的工资

**计算公式**：
```
摸鱼赚的钱 = 摸鱼时长(分钟) × (时薪 ÷ 60)
```

**示例**：
- 时薪 ¥50，上厕所 10分钟 → 赚了 ¥8.33
- 时薪 ¥50，下楼溜达 30分钟 → 赚了 ¥25

**摸鱼类型**：

| 类型 | 建议时长 | 备注 |
|-----|---------|------|
| 上厕所 | 5-15分钟 | 正常范围 |
| 下楼溜达 | 10-30分钟 | 透气时间 |
| 接水/泡咖啡 | 3-5分钟 | 小憩 |
| 和同事聊天 | 5-15分钟 | 社交时间 |
| 刷手机摸鱼 | 自定义 | ... |

**交互设计**：
- 开始摸鱼 → 结束摸鱼（自动计时）
- 或手动输入摸鱼时长
- 摸鱼类型选择
- 今日累计摸鱼时长 + 赚的钱

### 5.4 水电费配置

**默认选项**：

| 类型 | 电费 | 水费 |
|-----|------|------|
| 民水民电（住宅标准） | ¥0.6/度 | ¥5/吨 |
| 商业水电（办公标准） | ¥1.2/度 | ¥5.5/吨 |

**用户自定义**：
- 可手动输入当地水电费单价
- 不同城市水电费不同，支持自定义更准确

#### 充电记录方式：设备容量选项模式 ✅

**设计决策**：调研主流设备电池容量，提供5种左右选项

**主流设备电池容量调研**：

| 设备类型 | 电池容量（mAh） | 充满耗电（Wh） | 充满耗电（度） |
|---------|----------------|----------------|----------------|
| 小手机（iPhone标准版） | ~3500mAh | ~13Wh | ~0.013度 |
| 大手机（安卓旗舰/Pro版） | ~5000mAh | ~19Wh | ~0.019度 |
| 平板电脑 | ~8000mAh | ~30Wh | ~0.03度 |
| 笔记本电脑 | ~不适用 | ~50-80Wh | ~0.05-0.08度/小时 |
| 蓝牙耳机 | ~500mAh | ~2Wh | ~0.002度 |

> 💡 **计算公式**：Wh = mAh × 电压(V) ÷ 1000
> 手机电压约3.7V，笔记本按充电功率估算

**用户可选的5种充电选项**：

```

│                                         │
│  选择设备和充电情况：                     │
│                                         │
│  ○ 小手机充满一次（iPhone等，约0.01度）  │
│  ○ 大手机充满一次（安卓旗舰，约0.02度）  │
│  ○ 平板充满一次（约0.03度）              │
│  ○ 笔记本充电1小时（约0.05度）           │
│  ○ 蓝牙耳机充满一次（约0.002度）         │
│                                         │
│  充电次数：___ 次                        │
│                                         │
│  [确认记录]                             │
│                                         │

```

**备注**：
- 不追求精准，给用户一个合理估算即可
- 选项文案注明"约X度"，让用户知道这是估算值
- 可提供"自定义输入度数"给精确需求的用户

## 6. 隐私保护功能 (Privacy Lock)

> ⭐ **实际场景**：在公司使用时防止老板/同事路过看到薪资信息和"薅羊毛"界面

### 6.1 九宫格图案锁 ✅

**设计决策**：采用九宫格连线图案解锁（类似Android解锁图案）

**解锁界面设计**：
```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   ┌─────────────────────────────────────────────────────┐   │
│   │                                                     │   │
│   │                  🔒 已锁定                           │   │
│   │                                                     │   │
│   │              请绘制图案解锁                          │   │
│   │                                                     │   │
│   │         ┌───┬───┬───┐                              │   │
│   │         │ ● │ ● │ ● │                              │   │
│   │         ├───┼───┼───┤                              │   │
│   │         │ ● │ ● │ ● │                              │   │
│   │         ├───┼───┼───┤                              │   │
│   │         │ ● │ ● │ ● │                              │   │
│   │         └───┴───┴───┘                              │   │
│   │                                                     │   │
│   │         [忘记图案？重置所有数据]                     │   │
│   │                                                     │   │
│   └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**图案规则**：
- 至少连接4个点
- 支持复杂图案（最多9个点）
- 图案存储在本地（加密存储）
- 连线顺序即为解锁图案

---

### 6.2 锁定触发方式 ✅

**手动锁定**：
- 界面上有锁定按钮（如右上角🔒图标）
- 点击立即锁定，进入锁定界面

**自动锁定**：
- 用户可设置自动锁定间隔

**可选间隔选项**：

| 选项 | 适用场景 |
|-----|---------|
| 10秒 | 高度警惕，频繁锁定 |
| 15秒 | 中等警惕 |
| 30秒 | 平衡模式 |
| 1分钟 | 低警惕，偶尔有人经过 |
| 5分钟 | 很少有人打扰的环境 |

**自动锁定触发时机**：
- 用户无任何操作（无点击、无输入）超过设定时间
- 喝水提醒弹窗关闭后也算"无操作"开始计时
- 用户主动操作（点击、输入）会重置计时器

### 6.6 锁定功能开关 ✅

**设计决策**：锁定功能可选，不强制启用

**设置项**：
- 是否启用图案锁（开关）
- 设置/修改解锁图案
- 自动锁定间隔（下拉选择）
- 手动锁定按钮（界面右上角）

**未启用锁定时**：
- 无锁定按钮
- 无自动锁定
- 界面始终可见

### 6.7 设计细节确认 ✅

**锁定界面内容**：
- 显示工作时长即可（已工作X小时X分钟）
- 不显示薪资、不显示薅羊毛
- 简洁无害，老板看到不会觉得奇怪

**"有操作"定义**：
- 点击页面任意位置 → 重置自动锁定计时器
- 点击喝水提醒弹窗（包括选择喝水量、稍后、关闭）→ 重置计时器
- 任何输入操作 → 重置计时器

**"无操作"触发自动锁定**：
- 上述操作均未发生，且超过设定的间隔时间 → 自动锁定

## 设计原则（待讨论）
